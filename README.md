# bicycle-analysis

# 🚲 서울시 따릉이 이용 현황 분석 대시보드

본 프로젝트는 서울시 공공자전거(따릉이) 대여 이력 데이터와 기상 데이터를 결합하여 이용 패턴을 분석하고, 운영 효율화를 위한 데이터 기반의 인사이트를 제공하는 **Streamlit** 웹 대시보드입니다.

## 🔗 배포 주소

[서울시 따릉이 데이터 분석 대시보드 바로가기] https://yoonsol530bicycle-analysis-2026.streamlit.app/

## 🛠 사용 기술 (Tech Stack)

* **Language:** Python 3.x
* **Framework:** Streamlit
* **Database:** SQLite3
* **Library:** Pandas, Plotly (Interactive Visualization)
* **Deployment:** Streamlit Cloud

## 📊 주요 분석 내용

### 1. 연령대 및 성별 운동량 분석

* **분석 요약:** 성별/연령대별 이용 패턴을 분석하여 건강 증진 목적의 핵심 타겟 식별.
* **데이터 인사이트:** 4050 남성 그룹이 타 그룹 대비 1회 이용 시 평균 운동량이 압도적으로 높음. 단순 이동 수단보다 '고강도 유산소 운동' 목적으로 활용하고 있음을 시사함.
* **추천 액션:** 중장년층 남성 대상 '헬스케어 챌린지'나 '운동량 기반 리워드 프로그램' 기획 시 높은 참여율이 예상됨.

### 2. 기온별 이용 효율 분석 (Goldilocks Zone)

* **분석 요약:** 기온 변화에 따른 이용 건수와 이용 시간의 상관관계 시각화.
* **데이터 인사이트:** 15°C~25°C 사이 온화한 날씨에서 이용 효율이 정점을 찍는 '골디락스' 구간 확인됨. 30°C 초과 폭염이나 5°C 미만 혹한기에는 이용 시간은 급격히 감소함.
* **추천 액션:** 이용 효율이 높은 봄·가을철 집중 정비를 실시하고, 극한 기온 시기에는 단거리 이동 유도를 위한 '환승 할인' 강화가 필요함.

### 3. 스테이션 자전거 수급 불균형 분석

* **분석 요약:** 스테이션별 대여/반납 편차를 분석하여 자전거 부족 및 과적 지역 식별.
* **데이터 인사이트:** 특정 스테이션(빨간색)에서 대여가 집중되는 '순유출' 심화 현상으로 자전거 부족 리스크 존재함. 파란색 스테이션은 과적 적재로 인한 반납 장애 리스크가 높음.
* **추천 액션:** 실시간 수급 데이터를 기반으로 '재배치 우선순위'를 자동 설정하여 운영 차량 동선을 최적화하고, 유동적 인센티브 정책 도입을 권장함.

## 🚀 실행 방법 (How to Run)

로컬 환경에서 직접 실행하려면 아래 단계를 따르세요.

### 1. 리포지토리 클론

```bash
git clone https://github.com/사용자이름/bicycle-analysis.git
cd bicycle-analysis

```

### 2. 필요한 라이브러리 설치

```bash
pip install -r requirements.txt

```

### 3. 데이터베이스 파일 확인

`bicycle.db` 파일이 `app.py`와 같은 폴더에 있는지 확인합니다.

### 4. 대시보드 실행

```bash
streamlit run app.py

```

## 📁 프로젝트 구조

* `app.py`: Streamlit 메인 애플리케이션 코드
* `bicycle.db`: 분석용 SQLite 데이터베이스 파일
* `requirements.txt`: 프로젝트 의존성 라이브러리 목록
* `README.md`: 프로젝트 설명 문서

### 💡 프로젝트 특징

* **시각화**: Plotly의 이중 축(Dual-axis) 차트와 양방향 막대 차트를 활용해 복합적인 데이터 관계를 직관적으로 전달함.
* **안정적 성능**: `st.cache_resource`를 사용해 DB 연결 부하를 최적화함.
